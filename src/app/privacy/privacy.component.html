This Privacy Policy outlines how the Fitness Member Management App ("the App") handles data for personal trainers ("users") and their clients. By using the App, you agree to the practices described below.

1. Responsibilities

  • User Responsibilities: The App is designed for personal trainers to manage client data. Users are solely responsible for inputting, updating, and managing client information within the App.
  • Usage of the App: Users must ensure that the data entered is accurate and only used for client management purposes. The App should not be used for any illegal or unauthorized purposes.

2. Data Collected

The App collects the following client data fields:

  • Mandatory Fields:
    • Client ID (unique for each client)
    • Name
    • Date of Birth (DOB)
    • Gender (Male, Female, Unspecified)
    • Fitness Program (Fat Loss, Senior Fitness, Muscle Gain, Pre/Postnatal Fitness, Contest Preparation, Overall Fitness)
    • Contact Information (e.g., email, phone)
    • Joined Date
    • Ending Date
  • Optional Field: Special Health Notes
  • Client Classification: Clients may be marked as either VIP or Regular.

3. Transparency

  • All data is stored locally in the user’s browser, ensuring that no external servers or cloud platforms access client information.
  • Users are informed that data management (including deletion) is fully within their control.

4. Data Management

  • Storage Location: Client data is stored locally in the browser on the device used by the trainer. No cloud or external server storage is utilized.
  • Data Deletion: Users can permanently delete client data directly through the App’s interface. Deleted data cannot be recovered.

5. Data Security

The App currently does not include advanced security measures such as encryption or password protection. Users are advised to:

  • Secure their devices to prevent unauthorized access.
  • Regularly back up critical client data, as no in-app backup mechanism is provided.
  • Exercise caution when using shared or public devices.

6. Securing Data

  • The App ensures that all fields, except "Special Health Notes," require valid inputs to maintain data integrity.
  • A unique Client ID is required for every client, preventing duplication of entries.
  • Users are encouraged to delete outdated or unnecessary data to minimize potential risks.

7. User Rights

  • Users have full control over their clients’ data, including adding, editing, and deleting client records as needed.
  • No external entity can access or process client data.
